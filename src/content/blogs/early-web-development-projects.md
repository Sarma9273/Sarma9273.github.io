## Introduction

Every technical journey begins with smaller projects.

Before working on larger platforms such as CyberGPT, Sahaaya360, and my dynamic portfolio, I developed several frontend projects to understand how websites are structured and how users interact with them.

These projects may be technically smaller than my later work, but they played an important role in building my foundation.

## Landing-Page Project

The landing-page task focused on creating an attractive single-page website with clearly separated sections.

The project included components such as:

- Header and navigation

- Main banner

- Product or service description

- Feature sections

- Call-to-action buttons

- Footer

- Responsive spacing

This exercise helped me understand webpage structure, alignment, typography, layout hierarchy, and the importance of guiding the user’s attention.

## Temperature Converter

The temperature converter was one of my first interactive JavaScript projects.

The user could enter a temperature value, select or use a conversion type, and view the converted result.

The project strengthened my understanding of:

- Reading input values

- Working with formulas

- Updating webpage content

- Handling invalid input

- Connecting JavaScript with HTML

- Designing simple user interfaces

Although the logic was basic, it demonstrated how a static webpage can become an interactive tool.

## Early Personal Portfolio

I also developed an early personal portfolio to present my profile, education, skills, and projects.

This version introduced me to:

- Personal branding

- About sections

- Skills presentation

- Project cards

- Contact information

- Responsive navigation

- Website deployment

The project later became the foundation for my current portfolio-upgrade initiative.

## Interface Recreation Exercises

I worked on interface recreation and clone-style exercises to improve my ability to study an existing design and rebuild similar layouts using HTML and CSS.

These exercises helped me practise:

- Visual observation

- Spacing and alignment

- Card layouts

- Image placement

- Button styling

- Responsive behaviour

- Reusable design patterns

The objective was not to copy a brand for commercial use, but to understand how professional interfaces are constructed.

## Challenges Faced

During the beginning stage, even small issues such as centring an element, controlling image size, or making a page responsive required significant experimentation.

JavaScript errors also showed me the importance of checking input types, function calls, and browser-console messages.

These early challenges gradually improved my confidence and problem-solving approach.

## Learning Outcomes

These projects helped me build a foundation in:

- HTML structure

- CSS styling

- JavaScript basics

- Responsive design

- Forms and user inputs

- Browser debugging

- GitHub project publishing

- Basic interface design

More importantly, they taught me to learn through building rather than only watching tutorials.

## Future Scope

Some of these projects may be rebuilt with:

- React components

- Improved accessibility

- Better responsive behaviour

- Modern visual design

- Form validation

- Automated testing

- Cleaner code organisation

- TypeScript

- Improved deployment workflows

## Technology Stack

HTML, CSS, JavaScript, responsive web design, GitHub and browser developer tools.

## Key takeaway
These beginner projects formed the foundation of my web-development skills. They remind me that larger projects are created by gradually combining smaller concepts learned through consistent practice.
