## Introduction

While working as a Computer Teacher at Swaminarayan Gurukul International School, I observed that distributing digital learning materials individually to every laboratory computer was repetitive and inefficient.

Students required access to Python exercises, HTML and CSS examples, documents, images, assignments, and practical-lab resources. Copying the files manually to each system would consume time and create inconsistent versions.

To improve this process, I worked on a shared digital learning environment using the school computer-lab network.

## The Main Objective

The objective was to create a central location where students could access required files from their respective systems.

The shared location was presented through a common network drive, such as the Z: drive, so that students could open learning materials without searching through unrelated folders.

## Folder Organisation

The content was organised according to subject and activity.

A sample structure could include:

- Computer fundamentals

- Python programming

- HTML

- CSS

- Practical exercises

- Class-wise folders

- Assignments

- Images and media

- Teacher resources

- Completed student work

Consistent folder naming was important because school students should be able to locate a file without needing advanced technical knowledge.

## Educational Use

The shared environment supported activities such as:

- Opening Python examples

- Copying starter code

- Accessing HTML and CSS templates

- Reading lesson instructions

- Viewing class presentations

- Submitting practical work

- Sharing event-related content

- Conducting guided laboratory activities

This allowed more classroom time to be spent on teaching and practice instead of file transfer.

## Basic Security and Control

A school laboratory requires a balance between accessibility and control.

Students should be able to access the materials they need, but important teacher resources should not be modified accidentally.

The project therefore encouraged thinking about:

- Read-only folders

- Separate submission folders

- File permissions

- Backup copies

- Clear naming standards

- Prevention of accidental deletion

- Restriction of sensitive material

- Periodic content cleanup

## Challenges Faced

Different laboratory systems may have inconsistent network settings, stored credentials, permissions, or drive mappings.

A shared system is useful only when it works consistently across all machines. Therefore, troubleshooting connectivity and confirming access from multiple student computers formed an important part of the process.

Another challenge was keeping the content simple enough for younger learners. A technically perfect folder structure can still fail when students cannot understand where the files are located.

## Learning Outcomes

This practical work strengthened my understanding of:

- Local network sharing

- Drive mapping

- File permissions

- Computer-lab administration

- Digital-resource organisation

- Classroom workflow design

- Student-friendly technical support

- Basic data protection and backups

It also showed me how small technical improvements can make teaching more efficient.
